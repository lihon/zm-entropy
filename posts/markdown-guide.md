Markdown 是技术写作者最常用的标记语言。它语法简单，可读性强，能让你专注于内容而非格式。

## 基础语法

以下是 Markdown 最常用的语法：

### 标题

```
# 一级标题
## 二级标题
### 三级标题
```

### 强调

```
**加粗文本**
*斜体文本*
~~删除线~~
```

### 链接和图片

```
[链接文字](https://example.com)
![图片描述](image.png)
```

### 代码

行内代码用反引号 `` `code` ``，代码块用三个反引号：

```python
def hello():
    print("Hello, World!")
```

## 推荐工具

- **VS Code** — 内置 Markdown 预览，插件丰富
- **Typora** — 所见即所得的 Markdown 编辑器
- **Obsidian** — 双向链接，适合知识管理
- **Notion** — 团队协作，支持 Markdown 导出

## 写作建议

好的技术文章有几个共同特点：

1. **结构清晰** — 用小标题组织内容，让读者一目了然
2. **举例说明** — 抽象概念配合具体例子，降低理解门槛
3. **精简表达** — 删掉所有不必要的词句
4. **重视排版** — 合理使用空行、列表和代码块，提高可读性

> 写作即思考。写得清楚，才能想得清楚。

开始写吧。不需要完美，只需要开始。
